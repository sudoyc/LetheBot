LetheBot Architecture Flow

Gateway → Ingestion → Identity/Memory/Context → Pi → Evaluator/Policy → Executor → QQ, with audit and rollback

How it works

  • • Gateway only adapts QQ/NapCat protocol.
  • • Ingestion writes raw events before reasoning.
  • • ContextPack includes visible memory and minimal identity data.
  • • Pi proposes; policy/executor decides and acts.

Complex-case controls

  • • No mandatory group reply trigger.
  • • Suppressors downgrade risky or noisy actions.
  • • L0 policy excludes secrets and deleted memory.
  • • Tools require permissions, sandbox, audit, redaction.

Recovery and governance

  • • Memory source links, revisions, supersede, rollback.
  • • Proposals/admin digest for high risk cases.
  • • Users can request deletion, opt-out, unlink.
  • • Owner/admin can inspect redacted traces and /why.
