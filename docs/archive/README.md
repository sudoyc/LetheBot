# Documentation Archive

This directory stores historical LetheBot documents that are no longer part of the canonical development path.

Archive files may explain why decisions were made, but they are not completion evidence. Treat them as historical context only.

## Layout

- `completion/` — old completion reports, READY notes, and MVP claims.
- `loop/` — previous loop-state and long-running-agent checkpoint material.
- `plans/` — retired MVP/post-MVP phase plans and delivery checklists.
- `prompts/` — previous `/goal` prompts.
- `reviews/` — audits, baseline metrics, and direction reviews.
- `discussions/` — raw discussion logs and answer reviews.
- `artifacts/` — generated diagrams or HTML artifacts.
- `integrations/` — superseded integration notes.

Use current code, current command output, and canonical docs under `docs/` for implementation decisions.
